# 工程 6: 品質チェック（エディタ内で完結）

## 概要

エディタ上のコードとファイルだけで確認できる品質チェック項目をまとめます。ブラウザ操作や実機確認が必要な項目は含めません。

**重要**: この工程では、MCPログ（`coding-md/mcp-log/{page}/design-context.md`）に保存されたデザイン情報と、実装コードの整合性を確認します。

## 前提条件

- HTML / SCSS / JS の実装が完了していること
- 画像ファイルが配置済みであること
- MCPログ（`coding-md/mcp-log/{page}/design-context.md`）が存在すること

## MCPログの参照方法

品質チェックを実行する前に、対象ページのMCPログを確認してください：

```bash
# 例: トップページの場合
cat coding-md/mcp-log/top/design-context.md
```

MCPログには以下の情報が記録されています：
- **フォント情報**: font-family, font-weight の一覧
- **カラー情報**: CSS変数名とカラーコードの対応表
- **タイポグラフィ詳細**: セクションごとのfont-size, line-height, letter-spacing
- **画像一覧**: 使用されている画像のファイル名と保存先パス

## チェック項目

### 1. タイポグラフィ

**参照**: MCPログの「フォント情報」「タイポグラフィ詳細」セクション

- [ ] フォント指定がデザイン通りか（font-family）
  - MCPログの「フォント情報」テーブルと照合
  - 例: `Noto Sans JP` (日本語), `Sofia Pro` (英語) が正しく使用されているか
- [ ] フォントサイズがデザイン通りか（font-size）
  - MCPログの「タイポグラフィ詳細」セクションと照合
  - セクションごとのfont-sizeを確認（例: header 14px, MESSAGE見出し 32px）
- [ ] フォントウェイトがデザイン通りか（font-weight）
  - MCPログの「フォント情報」テーブルと照合
  - 例: Regular (400), Medium (500), Bold (700), Black (900) が正しく使用されているか
- [ ] 行間隔がデザイン通りか（line-height）
  - MCPログの「タイポグラフィ詳細」セクションと照合
  - 例: MESSAGE本文 line-height: 2.4, 見出し line-height: 1.6
- [ ] 文字間隔がデザイン通りか（letter-spacing / tracking）
  - MCPログの「タイポグラフィ詳細」セクションと照合
  - tracking値（px）を確認（例: ロゴ横テキスト tracking: 1.12px）

### 2. 色

**参照**: MCPログの「カラー情報」セクション

- [ ] テキスト色がデザイン通りか（color）
  - MCPログの「カラー情報」テーブルと照合
  - CSS変数（例: `var(--navy)`, `var(--black)`）が正しく使用されているか
  - カラーコード（例: `#042275`, `#232323`）が一致しているか
- [ ] 背景色がデザイン通りか（background-color）
  - MCPログの「カラー情報」テーブルと照合
  - 例: `--gray-bg: #f5f5f5`, `--blue-bg: #eef7ff` が正しく使用されているか
- [ ] ボーダー色がデザイン通りか（border-color）
  - MCPログの「カラー情報」テーブルと照合
  - 必要に応じて、デザインから読み取ったボーダー色を確認

### 3. 画像

**参照**: MCPログの「画像一覧」セクション

- [ ] 使用画像のファイル名が命名規則通りか
  - MCPログの「画像一覧」テーブルの「保存先パス」と照合
  - HTML内の画像パス（`src`属性）がMCPログの「保存先パス」と一致しているか
  - 例: `kv_bg.png`, `message_bg.png`, `interview_01.png` など
- [ ] 2倍書き出し前提の画像サイズ / @2x運用になっているか（命名・運用ルールの確認）
  - 画像ファイルの実際のサイズを確認（2倍解像度になっているか）
  - `resize-images-2x.sh` が実行されているか確認
- [ ] 画像の最適化（圧縮）処理の設定があるか（ビルド設定やスクリプト）
  - `gulpfile.js` やビルドスクリプトを確認
- [ ] WebP対応の出力設定があるか（ビルド設定やスクリプト）
  - `gulpfile.js` やビルドスクリプトを確認
- [ ] 画像ファイルが実際に存在するか
  - MCPログの「保存先パス」に記載されているファイルが存在するか確認
  - 例: `src/images/top/kv_bg.png` が存在するか

### 4. JS・アニメーション（コード上の確認）

#### 4-1. ホバー・インタラクション

- [ ] リンク・ボタンにホバー指定があるか（CSS / JS）
- [ ] ホバー指定が `@media (any-hover: hover)` でタッチデバイスと分離されているか
- [ ] フォーカス時のスタイル（`:focus-visible` など）が定義されているか

#### 4-2. アニメーション（CSS）

- [ ] `transition` / `animation` プロパティに適切な duration・easing が指定されているか
- [ ] `@media (prefers-reduced-motion: reduce)` でモーション低減対応がされているか（推奨）
- [ ] 不要な `will-change` が大量に指定されていないか（パフォーマンス観点）

#### 4-3. スクロール連動（JS）

- [ ] スクロールで発火するアニメーション（フェードイン等）が実装されているか
- [ ] 実装方法が `IntersectionObserver` など効率的な API を使用しているか（`scroll` イベント直接監視は避ける）
- [ ] スクロール対象の要素に適切なクラスやデータ属性が付与されているか

#### 4-4. プラグイン・ライブラリ

- [ ] Swiper / GSAP など使用プラグインのバージョンが `package.json` に固定されているか
- [ ] 初期化コードがページ読み込み後（`DOMContentLoaded` など）に実行されているか
- [ ] 不要になったプラグインが `package.json` や読み込み `<script>` から削除されているか

#### 4-5. JS の構造

- [ ] JS ファイル内に `console.log` / `debugger` などの開発用コードが残っていないか
- [ ] `alert` / `confirm` などの仮実装が残っていないか
- [ ] グローバルスコープの汚染がないか（IIFE やモジュールで囲まれているか）

### 5. meta / head

- [ ] title が設定されているか
- [ ] description が設定されているか
- [ ] keywords が設定されているか（必要時のみ）
- [ ] OGP 画像（og:image）が設定されているか
- [ ] favicon の設定があるか（link rel="icon"）

### 6. HTML / リンク

- [ ] リンク先が指定通りのパスになっているか（href）
- [ ] 見出し構造が適切か（h1 → h2 → h3...）
- [ ] 抜け漏れがあったら教えて

## チェック手順

1. **MCPログの確認**
   ```bash
   # 対象ページのMCPログを開く
   cat coding-md/mcp-log/{page}/design-context.md
   ```

2. **コードとの照合**
   - HTMLファイル: 画像パス、見出し構造、リンク先を確認
   - SCSSファイル: フォント、カラー、サイズを確認
   - 画像ファイル: ファイル名と存在確認

3. **不一致の記録**
   - MCPログとコードで不一致があった場合は、修正が必要です
   - 修正後、再度チェックを実施してください

## 注意事項

- ブラウザ操作が必要な確認（表示崩れ、レスポンシブ、コンソールエラー等）は別工程で実施
- 仕様書 / デザインから読み取れない項目は、実装責任者に確認してから記載・判断すること
- **MCPログに記録されていない情報は、Figmaから再取得するか、実装責任者に確認してください**
- MCPログの情報が古い場合は、最新の情報を取得してからチェックを実施してください
