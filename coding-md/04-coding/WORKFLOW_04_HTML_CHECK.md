# 工程 4 補足: HTML チェック

## 概要

HTML の構造、セマンティック、アクセシビリティなどをチェックします。

## 前提条件

- HTML ファイルが作成されていること

## チェック項目

### 1. HTML 構造

- [ ] DOCTYPE 宣言が正しいか
- [ ] `<html>` タグに `lang` 属性が設定されているか
- [ ] `<head>` 内のメタタグが適切か
- [ ] `<title>` が設定されているか

### 2. セマンティック HTML

- [ ] 適切な HTML タグが使用されているか
  - `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>` など
- [ ] 見出しタグ（`<h1>`-`<h6>`）が適切に使用されているか
- [ ] 見出しの階層が正しいか（`<h1>` → `<h2>` → `<h3>` の順）

### 3. リンク要素

- [ ] リンクとして機能すべき要素が `<a>` タグになっているか
- [ ] `<a>` タグに `href` 属性が設定されているか
- [ ] 外部リンクに `target="_blank"` と `rel="noopener noreferrer"` が設定されているか

### 4. 画像要素

- [ ] すべての `<img>` タグに `alt` 属性が記述されているか
- [ ] `alt` 属性の内容が具体的で適切か
- [ ] 装飾的な画像でも適切な説明が記述されているか

#### 固定レイアウトの画像（比率つきビジュアル）

デザインで表示枠の縦横比が決まっている画像は、**`aspect-ratio` は `<img>` ではなく親ラッパーに寄せる**（プロジェクト規約）。詳細・例外はリポジトリ直下の **`.cursor/rules/markup-bem-css.mdc`** の「画像のアスペクト比」を参照する。

- [ ] 比率を決める親要素が **`figure`、または `__figure` / `__media` などビジュアル用のクラスを付けたラッパー**になっているか
- [ ] **`news-card__img` のような「画像要素だが実体は空の `div` + CSS 背景」だけ**になっていないか（コンテンツ画像は **`figure` + `<img src>`** を優先する）
- [ ] `<figure>` を使う場合は **`margin: 0` は CSS 側でリセット**しうるため、セクションごとのスタイルで前提がぶれていないか（実装側ルールに従う）

**WHAT WE DO / NEWS の参考**：`<figure class="what-we-do__figure"><img … /></figure>`、`news-card__figure` と同等。

### 5. フォーム要素

- [ ] `<form>` タグが適切に使用されているか
- [ ] `<input>`, `<textarea>`, `<select>` に適切な `type` や `name` が設定されているか
- [ ] `<label>` タグが適切に使用されているか

### 6. アクセシビリティ

- [ ] キーボード操作でナビゲーション可能か
- [ ] 適切な ARIA 属性が使用されているか（必要に応じて）
- [ ] コントラスト比が適切か（これは CSS のチェック項目）

### 7. BEM 記法

- [ ] すべてのクラス名が BEM 記法に従っているか
- [ ] クラス名が適切に命名されているか

### 8. 構造の確認

- [ ] HTML の構造が論理的か
- [ ] ネストが適切か
- [ ] 閉じタグが正しく記述されているか

### 9. 共通パターンとクラス統一

**方針**: マークアップの繰り返しには共通クラスを優先し、セクション固有の差分だけを別クラスに分ける。判断の前提・原則は [ワークフローマスター](../WORKFLOW_MASTER.md) の「HTML / CSS クラス設計ルール」内 **「共通パターンの検出とクラス名の統一」** を参照する。

#### チェック項目

- [ ] **同じ構造が 2 箇所以上**あるブロック（見出しペア、見出し行＋「VIEW MORE」など）に、**共通のラッパー／共通のリンククラス**が付けられているか（セクション名だけ異なる `○○__head` が並列していないか）
- [ ] 共通パターンに対して、**役割が同じなのにクラス名だけがブロックごとに別名**になっていないか
- [ ] 共通クラスと **文脈クラスの分担**が妥当か（共通側に載せすぎず、セクション固有の余白・タイポは `○○__subtitle` などで足しているか）
- [ ] **無名のラッパー `div`** でレイアウトだけを挟んでいないか（レイアウト単位には役割が分かるクラスを付ける）

#### マークアップ例（このプロジェクトでの参考）

```html
<!-- 英題 + サブタイトル（複数セクションで共通） -->
<div class="section-heading">
  <h2 class="section-title">…</h2>
  <p class="section-subtitle section-subtitle--with-line news__subtitle">…</p>
</div>

<!-- 上記に VIEW MORE が横並びになるパターン（SERVICES / NEWS など） -->
<div class="section-heading-row">
  <div class="section-heading">…</div>
  <a class="section-heading-row__more" href="#">VIEW MORE</a>
</div>
```

※ クラス名はプロジェクトで統一したものに読み替える。

### 10. セクションと BEM の対応

**方針**: 名前付きセクション・一覧レイアウト・カード内テキストを、**ブロック／エレメントの対応が一目で分かる**ようにする。原則は [ワークフローマスター](../WORKFLOW_MASTER.md) の **「セクションと子要素の BEM 対応（名前付きブロック）」** を参照する。

#### チェック項目

- [ ] **名前付きのセクション**（サービス一覧・ニュースなど）の `<section>` に、その領域の **ブロック名**（例: `news`, `services`）が付いているか
- [ ] そのセクション内の **一覧・レイアウト用ラッパー**が **`{ブロック名}__grid`** など **親ブロックのエレメント**になっているか（セクション名だけ違う別ブロック名のラッパーを増やしていないか）
- [ ] **カード内で役割の違うテキスト**（タイトル・説明など）に **`{カードブロック}__title`** のような **エレメントクラス**があるか（タグ結合のみのスタイル指定になっていないか）
- [ ] **`○○__title` / `○○__text`** を使っているとき、親に **`○○` ブロック**があるか（例: `section-cta__title` と `section-cta`）

#### マークアップ例（NEWS ブロックとカード）

比率・トリミングは **親 `.news-card__figure`**（SCSS で `aspect-ratio`）、画像は `<img>` が親いっぱいに `object-fit: cover` で収まる形とする。

```html
<section class="section section--light news">
  <div class="container">
    <!-- 見出し行は共通パーツ section-heading-row を利用 -->
    <div class="news__grid">
      <article class="news-card">
        <figure class="news-card__figure">
          <img src="./public/assets/img/news-01.jpg" alt="" />
        </figure>
        <p class="news-tag">…</p>
        <p class="news-card__title">…</p>
        <time>…</time>
      </article>
    </div>
  </div>
</section>
```

## チェック手順

### 1. HTML バリデーターを使用

```bash
# W3C HTMLバリデーターを使用
# https://validator.w3.org/ でチェック
```

### 2. エディタのリンター機能を使用

- VS Code などのエディタの HTML リンター機能を使用
- エラーや警告を確認

### 3. 手動チェック

- 上記のチェック項目を 1 つずつ確認
- 問題があれば修正

### 4. ブラウザで確認

- 複数のブラウザで表示を確認
- 開発者ツールでエラーがないか確認

## 修正例

### セマンティック HTML の修正

```html
<!-- 修正前 -->
<div class="header">...</div>
<div class="content">...</div>
<div class="footer">...</div>

<!-- 修正後 -->
<header class="header">...</header>
<main class="content">...</main>
<footer class="footer">...</footer>
```

### リンク要素の修正

```html
<!-- 修正前 -->
<div class="news__item" onclick="...">
  <p class="news__item-title">ニュースタイトル</p>
</div>

<!-- 修正後 -->
<a href="#" class="news__item-link">
  <p class="news__item-title">ニュースタイトル</p>
</a>
```

### alt 属性の追加

```html
<!-- 修正前 -->
<img src="{画像パス}/{ファイル名}" class="{クラス名}" />

<!-- 修正後 -->
<img src="{画像パス}/{ファイル名}" alt="{具体的な説明}" class="{クラス名}" />
```

## 注意事項

- セマンティック HTML を必ず使用すること
- リンク要素は必ず `<a>` タグを使用すること
- alt 属性は必ず記述すること
- アクセシビリティを考慮した実装を心がけること
- **繰り返しパターンは共通クラスを検討し**（マスター参照）、セクション名だけが違う冗長なラッパー名を増やしすぎないこと
- **名前付きセクションは `<section>` にブロック名を付け**、一覧ラッパーは **`{block}__grid`**、カード内のタイトルなどは **`{card}__title`** のように **BEM で対応関係が分かる**ようにすること
- **カード・セクションの固定比率ビジュアル**は、「### 4. 画像要素」の固定レイアウト画像ルールと `.cursor/rules/markup-bem-css.mdc` に従う
